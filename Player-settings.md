# Player Settings

Player settings should support both comfort and situational play. Because the game includes long travel, combat, building, screenshots, social spaces, and large-scale events, players should be able to save multiple named profiles and switch between them quickly.

## Setting Profiles

A profile stores the same kinds of options players expect from a modern PC game, but grouped for different use cases. A combat profile might reduce visual clutter, raise important audio cues, and prioritize performance. A cinematic profile might increase graphics quality, widen ambience, and hide optional UI elements for screenshots. A default profile should remain comfortable for general exploration and settlement life.

Profiles should not create competitive advantages beyond letting players choose readable, accessible settings. Any setting that affects fairness, targeting, visibility of hidden enemies, or server-authoritative gameplay should be constrained by the normal rules of the game.

## Audio Layers

Audio settings should be divided into clear layers so players can tune the mix to their needs. Useful layers include master volume, music, ambience, weather, creature cues, combat impacts, voice chat, interface sounds, crafting, settlement ambience, and warning signals. Important danger cues should remain understandable even at lower volumes or through accessibility alternatives.
